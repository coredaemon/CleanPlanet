# MEDIA_GUIDE.md

## Текущая структура

В корне проекта находятся каталоги:

```text
01/
02/
```

### Каталог `01`

В нём лежат уже склеенные изображения «до/после».

Пример:

```text
09_01-02.jpg
```

- `09` — номер набора;
- `01-02` — в одном файле объединены «до» и «после».

Такие изображения считаются готовыми коллажами. Их нельзя автоматически разделять.

### Каталог `02`

В нём лежат отдельные пары:

```text
01_01.jpg
01_02.jpg
```

- первая часть — номер пары;
- `_01` — «до»;
- `_02` — «после».

Примеры:

```text
01_01.jpg + 01_02.jpg
02_01.jpg + 02_02.jpg
03_01.jpg + 03_02.jpg
```

## Исходники

Исходные файлы нельзя изменять, удалять или переименовывать:

```text
01/*.jpg
02/*.jpg
```

Производные файлы создавать отдельно:

```text
public/images/projects/
├── collages/
├── before/
├── after/
└── thumbnails/
```

## Имена production-файлов

Коллаж:

```text
project-09-before-after.webp
```

Пара:

```text
project-01-before.webp
project-01-after.webp
```

## Форматы и размеры

Основной формат — WebP. AVIF допустим с fallback.

- крупные изображения: 1600–2000 px;
- галерея: 1200–1600 px;
- карточки: 720–960 px;
- миниатюры: 320–480 px.

Не увеличивать выше исходного разрешения.

## До/после

Допустимые варианты:

1. Две фотографии рядом.
2. Интерактивный слайдер — только для отдельных пар из `02`.
3. Готовый коллаж — для файлов из `01`.

Не использовать слайдер для уже склеенных файлов.

## Кадрирование

- сохранять одинаковое соотношение сторон;
- не искажать результат;
- не ретушировать загрязнение;
- не усиливать искусственно разницу между «до» и «после».

## Alt

Примеры:

```text
Кухня до генеральной уборки
Кухня после генеральной уборки
Результат уборки ванной комнаты до и после
```

## Конфиденциальность

Перед публикацией проверить лица, документы, адреса, номера квартир, семейные фото, экраны, автомобильные номера и другие персональные данные.

## Запреты для агентов

Агентам запрещено:

- удалять исходники;
- переименовывать исходники;
- менять каталоги `01` и `02`;
- путать `_01` и `_02`;
- автоматически публиковать всё;
- придумывать описания кейсов;
- ретушировать результат;
- накладывать логотип без команды.

## Импорт

Скрипт обработки должен:

1. считать файлы в `01` готовыми коллажами;
2. сгруппировать `02` по номеру;
3. определить `_01` как before;
4. определить `_02` как after;
5. вывести неполные пары;
6. создать производные WebP;
7. не менять исходники;
8. сформировать JSON-манифест.
