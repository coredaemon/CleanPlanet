# Замечания по сайту (черновик, для последующей проработки)

> Сюда складываем всё, что заметили при просмотре превью (https://coredaemon.github.io/CleanPlanet/).
> Потом пройдёмся по списку и решим, что и как чинить.

> **СТАТУС:** все пункты структурированы в спеку для Codex → `CODEX_FIX_UX2.md`.
> **Найдена корневая причина** нечитаемой шапки и «сломанного» лайтбокса: классы прозрачности вне шкалы Tailwind v3 (`bg-white/92`, `bg-black/82`, `bg-white/96`, `white/12`, `/78`, `/65`) молча не генерируются → элементы без фона. Подтверждено по собранному CSS.

## Список

- [ ] **`/projects/` — лайтбокс галереи сломан.**
  - На десктопе: открытое фото занимает почти весь экран (огромное), стрелка «вперёд»/пролистывание на следующее фото не работает — застревает на первой.
  - На мобильном: выглядит криво — лайтбокс маленький, наезжает на cookie-баннер и нижнюю панель (Позвонить/Написать/Рассчитать), всё перекрывает друг друга.
  - Нужно переделать: либо нормальный полноэкранный слайдер с рабочим пролистыванием, либо другой подход к отображению галереи на странице.

- [ ] **`/services/` (каталог услуг) — две формы заявки на одной странице.**
  - Калькулятор сверху с кнопками «Оставить заявку» / «Отправить фотографии» + отдельная полноценная форма «Помочь выбрать услугу» снизу — по сути дубль.
  - Должна остаться только ОДНА форма на странице (просто «Оставьте заявку» / расчёт), без второй дублирующей формы ниже.

- [ ] **`/services/` — секция «Предварительный калькулятор» (текстовая, не рабочая) — удалить.**
  - На странице есть текстовый блок «Предварительный калькулятор» с нумерованным списком параметров и результатом «Предварительная стоимость: [РАСЧЁТНЫЙ ДИАПАЗОН]» — это не настоящий калькулятор, просто текст-заглушка, смысла в нём нет.
  - Убрать эту секцию целиком с этой страницы.

- [ ] **Шапка сайта — плохая читаемость при прокрутке (липкая шапка).**
  - Во время скролла в некоторых местах пункты меню и телефон в шапке становятся плохо различимы/нечитаемы (см. скриншот — текст в шапке смазан/сливается с фоном).
  - Нужно поправить фон/контраст липкой шапки, чтобы меню и телефон были всегда чётко читаемы при любой прокрутке.

- [ ] **Главная — hero-блок: 3 кнопки («Рассчитать стоимость» / «Оставить заявку» / «Отправить фотографии») — слишком много, все ведут в одно место.**
  - Оставить одну кнопку — «Оставить заявку».

- [ ] **Дублирующие друг друга секции с формой/расчётом — по всему сайту, не только на главной.**
  - На главной по факту два блока с одним и тем же смыслом: калькулятор «Уточните параметры уборки» и отдельная секция «Получить предварительный расчёт» — это одно и то же (заявка/звонок), просто разложено на разные секции.
  - Нужно переработать в ОДНУ основную секцию приёма заявки (с параметрами уборки внутри), а не размножать похожие блоки на одной странице.
  - Пройтись по ВСЕМ страницам сайта и проверить на такое же дублирование форм/расчётов (см. также пункты выше про `/services/`).

- [ ] **Единая форма заявки для ВСЕГО сайта (заменяет все текущие варианты форм).**

  Одна и та же форма должна использоваться на всех страницах — без вариаций, без вторых форм рядом. Состав полей:
  1. **Имя** — как обращаться (обязательно).
  2. **Телефон** — обязательно, с **маской ввода** (сейчас можно вбить бесконечное количество цифр — это поправить, маска должна нормально «водиться» при вводе).
  3. **Услуга** — выпадающий список (select), аккуратный, без чекбоксов. Внутри можно выбрать одну из нескольких услуг.
  4. **Локация** — вместо поля «Площадь или адрес» (люди не хотят сразу давать адрес): переключатель **«Москва» / «За МКАД»**. Если выбрано «За МКАД» — показывать доп.поле «Примерное расстояние от МКАД».
  5. **Срочно?** — чекбокс/переключатель «Да / Нет» (нужна ли срочная уборка).
  6. **Фотографии** — по желанию, подпись коротко: **«Фотографии (по желанию, до 5 изображений)»**. Убрать из текста упоминания форматов файлов и автоматического сжатия — это лишнее, пользователю не нужно.
  7. **Комментарий** — свободное поле, что нужно сделать.
  8. **Согласие на обработку персональных данных** — чекбокс по 152-ФЗ (как сейчас), не отмечен по умолчанию.

  Больше никаких полей не нужно — этого достаточно.

  **Поведение формы на странице:**
  - Форма внизу страницы (в подвальной секции) — как обычно, встроена в страницу.
  - НО если человек нажимает кнопку заявки/расчёта где-то **выше по странице** (не в самом низу) — форма должна открываться **как всплывающий поп-ап по центру экрана**, а не скроллить/прыгать к блоку внизу страницы. Никаких резких «прыжков» на другую часть страницы или на другую страницу.
  - Кнопка в шапке сайта (сейчас «Рассчитать») — по нажатию тоже должна открывать этот поп-ап с формой, а не скроллить к секции с формой на странице.

  По сути это заменяет собой всё, что было раньше: разные виды форм (обычная заявка / фото-форма и т.п.) — теперь только ОДНА универсальная форма и ОДИН поп-ап-механизм для неё везде на сайте.
